# COGNUS: Visual Identity Guide (Executive Summary)

This document summarizes the institutional visual identity of COGNUS and points to the operational design system used by the frontend.

## 1) Visual Concept

The COGNUS identity uses a **constellation and network** metaphor:

- nodes represent organizations and components;
- connections represent multi-organization coordination;
- visual highlights represent decisions, evidence, and operational state.

## 2) Institutional Base Palette

- Night blue: `#1B2559` for institutional surfaces
- Action blue: `#2F6BFF` for primary actions and links
- Network green: `#2CB6A6` for connectivity and health
- Evidence violet: `#7C3AED` for governance and emphasis

Semantic states:

- success: `#16A34A`
- info: `#0284C7`
- warning: `#D97706`
- failure or blocked: `#DC2626`

## 3) Recommended Typography

- Interface and content: `Inter`
- IDs, hashes, and logs: `JetBrains Mono`

## 4) Usage Guidance

- Product frontend: use `docs/frontend/COGNUS_identidade_visual.md` as the normative source.
- Technical documents should keep a sober visual language, clear readability, and consistent status semantics.
- Topology views should prioritize operational legibility over decorative effects.
